# Visual Studio Code Ziper

VS Code のエクスプローラーのコンテキストメニューから手軽に ZIP 圧縮・解凍を行える拡張機能です。プロジェクトの実装状態に合わせて README を更新しています。

---

## 主な機能

- エクスプローラーでフォルダーまたはファイルを右クリックして `ZIP` を実行
- ZIP ファイルを右クリックして `Unzip` を実行
- フォルダー構造を完全に保持（空フォルダー含む）
- 圧縮済みファイル（例: .jpg, .png, .mp4 など）は `STORE`（無圧縮）で格納し、サイズ増加を防止
- 圧縮後にストリームベースの整合性検証を実施（CRC32 とサイズの確認）
- Zip Slip 攻撃に対する展開先パス検証
- 進捗表示と操作中のキャンセル/エラーハンドリング

---

## 使い方

### 圧縮 (ZIP 作成)
1. エクスプローラーで対象を右クリック
2. メニューから `ZIP` を選択
3. 進捗通知が表示され、完了後に保存先に `.zip` が作成されます

### 解凍 (Unzip)
1. ZIP ファイルを右クリック
2. `Unzip` を選択
3. 同じディレクトリに展開されます（既存のファイルがある場合は確認ダイアログ）

---

## 圧縮の自動判定

内部で拡張子ベースの判定を行い、既に圧縮済みのファイルは `STORE`（無圧縮）で格納します。これにより、逆にサイズが増えることを防ぎます。

例:
- テキスト系 (.txt, .json, .xml) → DEFLATE
- 画像/動画/音声/他のアーカイブ (.jpg, .mp4, .mp3, .zip, .7z 等) → STORE

---

## セキュリティと整合性

- Zip Slip 防止: 展開前に対象パスが意図した範囲内か検査します。
- 整合性検証: 現在はストリームベースで各エントリの CRC32 と展開サイズを検証します。大きなアーカイブでもメモリ消費を抑えつつ検証します。
- エラー時クリーンアップ: 作成や展開が途中で失敗した場合、壊れた中間ファイルを削除します。

## インストール

### 自動インストーラ
- `Visual-Studio-Code-Ziper(VSCZ)/installer.cmd` を使うと、同じフォルダにある `.vsix` を `code --install-extension` でインストールします（`code` が PATH にあること）。

### 手動
1. VS Code を開く
2. `Ctrl+Shift+P` → `Extensions: Install from VSIX...` を選択
3. 本リポジトリの `visual-studio-code-ziper-0.0.2.vsix` を指定してインストール

注: ローカルで VSIX をインストールした場合、VS Code のバージョンや挙動により拡張機能の詳細ページへ自動的に README が表示されない場合があります。本プロジェクトではパッケージング時に `build/inject-readme.js` を使って `README.md` を VSIX に通常ファイルとして含める処理を行っています。

---

## 開発者向け情報

- ビルド: `npm run compile`（`config` ディレクトリのスクリプトに依存）
- パッケージ: `npm run package`（VSIX を生成し、`build/inject-readme.js` で README を注入します）
- バンドル: `esbuild` を使用して `dist/extension.js` を生成

依存: `archiver`（作成）、`yauzl`（ストリーミング検証）などを使用しています。
